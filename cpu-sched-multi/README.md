# 概述

欢迎使用 multi.py，一个基础的多 CPU 调度模拟器。此模拟器有许多功能可供探索，请仔细阅读！当然，如果你懒，不读也行。但考试来了你就知道...

要运行模拟器，你只需输入：

```
提示符> ./multi.py
```

这将使用一些随机作业运行模拟器。

在我们深入了解这些细节之前，让我们先检查一下此类模拟的基础知识。

在默认模式下，系统中有一个或多个 CPU（由 -n 标志指定）。因此，要在你的模拟中运行 4 个 CPU，请键入：

```
提示符> ./multi.py -n 4
```

每个 CPU 都有一个缓存，可以保存一个或多个运行进程的重要数据。每个 CPU 缓存的大小由 -M 标志设置。因此，要在你的 4-CPU 系统上使每个缓存大小为 '100'，请运行：

```
提示符> ./multi.py -n 4 -M 100
```

要运行模拟，你需要一些要调度的作业。有两种方法可以做到这一点。第一种是让系统为你创建一些具有随机特征的作业（这是默认设置，即，如果你不指定任何内容，你会得到这个）；还有一些控制可以（在某种程度上）控制随机生成的作业的性质，下文将进一步描述。第二种是为系统精确指定一个要调度的作业列表；这也将在下文更详细地描述。

每个作业都有两个特征。第一个是它的"运行时间"（它将运行多少个时间单位）。第二个是它的"工作集大小"（它需要多少缓存空间才能高效运行）。如果你正在随机生成作业，你可以通过使用 -R（最大运行时间标志）和 -W（最大工作集大小标志）来控制这些值的范围；然后随机生成器将生成不大于这些值的值。

如果你是手动指定作业，则可以使用 -L 标志显式设置每个作业的这些值。例如，如果你想运行两个作业，每个作业的运行时间都为 100，但工作集大小分别为 50 和 150，你可以这样做：

```
提示符> ./multi.py -n 4 -M 100 -L 1:100:50,2:100:150
```

请注意，你为每个作业分配了一个名称，第一个作业为 '1'，第二个作业为 '2'。当作业是自动生成时，名称是自动分配的（只使用数字）。

作业在特定 CPU 上运行的效果如何，取决于该 CPU 的缓存当前是否持有给定作业的工作集。如果没有，作业运行缓慢，这意味着每个时钟滴答，只有 1 个滴答从其剩余时间中减去。这是该作业的缓存处于"冷"状态的模式（即，它尚未包含该作业的工作集）。然而，如果作业之前在该 CPU 上运行了"足够长的时间"，那么该 CPU 缓存现在处于"热"状态，作业将执行得更快。有多快，你可能会问？好吧，这取决于 -r 标志的值，即"预热速率"。这里，默认情况下大约是 2 倍，但你可以根据需要更改它。

缓存需要多长时间才能预热，你可能会问？好吧，这也是由一个标志设置的，在这种情况下，是 -w 标志，它设置了"预热时间"。默认情况下，大约是 10 个时间单位。因此，如果一个作业运行了 10 个时间单位，该 CPU 上的缓存就会变热，然后作业开始运行得更快。当然，所有这些都只是对真实系统工作原理的粗略近似，但这就是模拟的魅力所在，对吧？

所以现在我们有了 CPU，每个都有缓存，以及一种指定作业的方法。还缺什么？啊哈，你说，调度策略！你说得对。真是个好学、勤奋的好学生！

第一个（默认）策略很简单：一个集中式调度队列，以轮转方式将作业分配给空闲 CPU。第二个是每个 CPU 调度队列方法（用 -p 打开），其中作业被分配给 N 个调度队列之一（每个 CPU 一个）；在这种方法中，空闲 CPU 会（偶尔）窥视其他 CPU 的队列并窃取一个作业，以改善负载平衡。这样做的频率由一个"窥视"间隔设置（由 -P 标志设置）。

有了这些基本的理解，你现在应该能够完成作业，并研究不同的调度方法。要查看此模拟器的完整选项列表，只需键入：

```
提示符> ./multi.py -h
用法: multi.py [选项]

选项:
  -h, --help            显示此帮助信息并退出
  -s SEED, --seed=SEED  随机种子
  -j JOB_NUM, --job_num=JOB_NUM
                        系统中的作业数量
  -R MAX_RUN, --max_run=MAX_RUN
                        随机生成作业的最大运行时间
  -W MAX_WSET, --max_wset=MAX_WSET
                        随机生成作业的最大工作集
  -L JOB_LIST, --job_list=JOB_LIST
                        提供以逗号分隔的作业列表
                        格式为 作业名:运行时间:工作集大小 (例如，
                        a:10:100,b:10:50 表示 2 个作业，运行时间都为 10，
                        第一个作业 (a) 的工作集大小=100，第二个作业 (b)
                        的工作集大小=50)
  -p, --per_cpu_queues  每个 CPU 的调度队列（而不是一个）
  -A AFFINITY, --affinity=AFFINITY
                        一个作业列表，指定它们可以在哪些 CPU 上运行 (例如，
                        a:0.1.2,b:0.1 允许作业 a 在 CPU 0,1,2 上运行，但 b
                        只能在 CPU 0 和 1 上运行
  -n NUM_CPUS, --num_cpus=NUM_CPUS
                        CPU 数量
  -q TIME_SLICE, --quantum=TIME_SLICE
                        时间片长度
  -P PEEK_INTERVAL, --peek_interval=PEEK_INTERVAL
                        对于每个 CPU 的调度，窥视其他调度队列的频率；
                        0 关闭此功能
  -w WARMUP_TIME, --warmup_time=WARMUP_TIME
                        预热缓存所需的时间
  -r WARM_RATE, --warm_rate=WARM_RATE
                        使用热缓存运行时快多少倍
  -M CACHE_SIZE, --cache_size=CACHE_SIZE
                        缓存大小
  -o, --rand_order      CPU 以随机顺序获取作业
  -t, --trace           启用基本跟踪（显示哪些作业被调度）
  -T, --trace_time_left
                        跟踪每个作业的剩余时间
  -C, --trace_cache     也跟踪缓存状态（热/冷）
  -S, --trace_sched     跟踪调度器状态
  -c, --compute         为我计算答案
```

可能最好你得了解一些跟踪选项（如 -t, -T, -C 和 -S）。尝试使用这些选项，以更好地了解调度器和系统正在做什么。
