# 概述

这个程序允许你查看在一个拥有基址和界限寄存器的系统中，地址转换是如何执行的。和之前一样，运行程序来测试你对基址和界限的理解分为两个步骤。首先，不使用 -c 标志运行，以生成一组转换，并查看你自己是否能正确执行地址转换。然后，完成后，使用 -c 标志运行以检查你的答案。

在本作业中，我们将假设一个与我们经典的地址空间略有不同的地址空间，其中堆和栈分别位于地址空间的两端。相反，我们将假设地址空间有一个代码段，然后是一个固定大小（较小）的栈，紧接着是一个向下增长的堆，看起来如下图所示。在这种配置中，只有一种增长方向，即向地址空间的更高区域。

```
-------------- 0KB
  |    代码     |
  -------------- 2KB
  |    栈      |
  -------------- 4KB
  |     堆     |
  |     |      |
  |     v      |
  -------------- 7KB
  |   (空闲)   |
  |     ...    |
```

在图中，界限寄存器将被设置为 7 KB，因为这代表了地址空间的末尾。对界限内的任何地址的引用将被视为合法；超出此值的引用是越界的，因此硬件将引发异常。

要使用默认标志运行，在命令行输入 `relocation.py`。结果应该类似于这样：

```
提示符> ./relocation.py
...
基址和界限寄存器信息:

  基址  : 0x00003082 (十进制 12418)
  界限  : 472

虚拟地址跟踪
  VA  0: 0x01ae (十进制:430) -> 物理地址 或 违规?
  VA  1: 0x0109 (十进制:265) -> 物理地址 或 违规?
  VA  2: 0x020b (十进制:523) -> 物理地址 或 违规?
  VA  3: 0x019e (十进制:414) -> 物理地址 或 违规?
  VA  4: 0x0322 (十进制:802) -> 物理地址 或 违规?
```

对于每个虚拟地址，要么写下它转换到的物理地址，要么写下它是一个越界地址（分段违规）。对于这个问题，你应该假设一个给定大小的简单虚拟地址空间。

如你所见，作业只是生成随机的虚拟地址。对于每个地址，你应该判断它是否在界限内，如果是，则确定它转换到哪个物理地址。使用 -c 标志（“为我计算”标志）运行，会给出这些转换的结果，即它们是有效还是无效，如果有效，则给出生成的物理地址。为了方便起见，所有数字都以十六进制和十进制给出。

```
提示符> ./relocation.py -c
...
虚拟地址跟踪
  VA  0: 0x01ae (十进制:430) -> 有效: 0x00003230 (十进制:12848)
  VA  1: 0x0109 (十进制:265) -> 有效: 0x0000318b (十进制:12683)
  VA  2: 0x020b (十进制:523) -> 分段违规
  VA  3: 0x019e (十进制:414) -> 有效: 0x00003220 (十进制:12832)
  VA  4: 0x0322 (十进制:802) -> 分段违规
```

在基址为 12418（十进制）的情况下，地址 430 在界限内（即，它小于界限寄存器值 472），因此转换为 430 加上 12418，即 12848。上面显示的一些地址是越界的（523, 802），因为它们超出了界限。非常简单，不是吗？确实，这就是基址和界限的美妙之处之一：它实在太简单了！

你可以使用一些标志来更好地控制程序的运行：

```
提示符> ./relocation.py -h
用法: relocation.py [选项]

选项:
  -h, --help            显示此帮助信息并退出
  -s SEED, --seed=SEED  随机种子
  -a ASIZE, --asize=ASIZE
                        地址空间大小 (例如: 16, 64k, 32m)
  -p PSIZE, --physmem=PSIZE
                        物理内存大小 (例如: 16, 64k)
  -n NUM, --addresses=NUM
                        要生成的虚拟地址数量
  -b BASE, --b=BASE     基址寄存器的值
  -l LIMIT, --l=LIMIT   界限寄存器的值
  -c, --compute         为我计算答案
```

特别是，你可以控制虚拟地址空间的大小 (-a)、物理内存的大小 (-p)、要生成的虚拟地址数量 (-n)，以及此进程的基址和界限寄存器的值（分别为 -b 和 -l）。
