# 概述

这份作业通过使用 `disk.py`来帮助你熟悉现代硬盘驱动器的工作原理。它有许多不同的选项，并且与大多数其他模拟器不同，它有一个图形化动画器，可以精确地向你展示磁盘运行时的情况。

注意：还有一个实验性程序 `disk-precise.py`。这个版本的模拟器使用 Python 的 Decimal 包进行精确的浮点计算，因此在某些极端情况下能提供比 `disk.py`稍好一些的结果。不过，它没有被非常仔细地测试过，所以请自行谨慎使用。

我们先从一个简单的例子开始。要运行模拟器并计算一些基本的寻道、旋转和传输时间，你首先需要给模拟器提供一个请求列表。这可以通过指定确切的请求，或者让模拟器随机生成一些请求来实现。

我们首先自己指定一个请求列表。先处理一个请求：

```
prompt> disk.py -a 10
```

此时你会看到：

```
...
REQUESTS [br '10']

对于上述请求，计算寻道、旋转和传输时间。
使用 -c 或图形模式 (-G) 查看答案。
```

要计算这个请求的寻道、旋转和传输时间，你需要了解更多关于扇区布局、磁头起始位置等的信息。要查看大部分此类信息，请在图形模式下运行模拟器 (-G)：

```
prompt> ./disk.py -a 10 -G
```

此时，应该会出现一个窗口，显示我们的简易磁盘。磁头位于外圈磁道上，在第6扇区中间。如你所见，扇区10（我们的示例扇区）在同一个磁道上，大约在圆周的三分之一处。旋转方向是逆时针。要运行模拟，请在模拟器窗口高亮时按 "s" 键。

当模拟完成时，你应该能看到磁盘花费了105个时间单位旋转，30个时间单位传输来访问扇区10，没有寻道时间。按 "q" 关闭模拟器窗口。

要计算这个（而不是仅仅运行模拟），你需要了解关于磁盘的一些细节。首先，默认旋转速度设置为每时间单位1度。因此，完成一次完整的旋转需要360个时间单位。其次，传输在扇区之间的中点开始和结束。因此，要读取扇区10，传输在扇区9和10之间开始，在扇区10和11之间结束。最后，在默认磁盘中，每个磁道有12个扇区，这意味着每个扇区占据30度的旋转空间。因此，读取一个扇区需要30个时间单位（考虑到我们的默认旋转速度）。

掌握了这些信息，你现在应该能够计算访问扇区10的寻道、旋转和传输时间了。因为磁头起始位置与扇区10在同一磁道，所以寻道时间为0。因为磁盘以每时间单位1度的速度旋转，所以到达扇区10的开始点（扇区9和10之间的中点）需要105个时间单位（注意，到达扇区9的中点恰好是90度，再额外15度到达中点点）。最后，传输扇区需要30个时间单位。

现在让我们做一个稍微复杂一点的例子：

```
prompt> ./disk.py -a 10,11 -G
```

在这种情况下，我们要传输两个扇区，10和11。这会花费多长时间？在运行模拟之前先猜一下！

正如你可能猜到的，这次模拟只多花了30个时间单位，用来传输下一个扇区11。因此，寻道和旋转时间保持不变，但请求的传输时间加倍了。事实上，你可以在模拟器窗口顶部看到这些总和；它们也会在控制台打印出来，如下所示：

```
...
扇区:  10  寻道:  0  旋转:105  传输: 30  总计: 135
扇区:  11  寻道:  0  旋转:  0  传输: 30  总计:  30
总计        寻道:  0  旋转:105  传输: 60  总计: 165
```

现在让我们做一个包含寻道的例子。尝试以下请求集：

```
prompt> disk.py -a 10,18 -G
```

要计算这将花费多长时间，你需要知道一次寻道需要多久。默认情况下，磁道之间的距离是40个距离单位，默认寻道速率是每单位时间1个距离单位。因此，从外圈磁道到中间磁道的寻道需要40个时间单位。

你还需要知道调度策略。不过默认是FIFO，所以现在你可以假设处理顺序与通过 `-a`标志指定的列表匹配来计算请求时间。

要计算磁盘服务这些请求需要多长时间，我们首先计算访问扇区10所需的时间，从上文可知是135个时间单位（105旋转，30传输）。一旦这个请求完成，磁盘开始寻道到扇区18所在的中间磁道，花费40个时间单位。然后磁盘旋转到扇区18，并传输30个时间单位，从而完成模拟。但这最后的旋转需要多长时间呢？

要计算扇区18的旋转延迟，首先要计算假设寻道成本为零的情况下，磁盘从扇区10的访问结束旋转到扇区18的访问开始需要多长时间。从模拟器中你可以看到，外圈磁道上的扇区10与中间磁道上的扇区22对齐，并且有7个扇区将22与18隔开（23, 12, 13, 14, 15, 16, 和 17，因为磁盘逆时针旋转）。旋转经过7个扇区需要210个时间单位（每个扇区30）。然而，这部分旋转的最初部分实际上花费在了寻道到中间磁道上，耗时40个时间单位。因此，访问扇区18的实际旋转延迟是210减去40，即170个时间单位。运行模拟器自己看看；注意你可以不带图形，用 "-c" 标志运行，只看结果不看图形。

```
prompt> ./disk.py -a 10,18 -c
...
扇区:  10  寻道:  0  旋转:105  传输: 30  总计: 135
扇区:  18  寻道: 40  旋转:170  传输: 30  总计: 240
总计        寻道: 40  旋转:275  传输: 60  总计: 375
```
