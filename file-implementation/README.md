# 概述

使用此工具 `vsfs.py`来研究文件系统状态如何随着各种操作的发生而变化。文件系统以一个空状态开始，只有一个根目录。随着模拟的进行，执行各种操作，从而逐步改变文件系统的磁盘状态。

可能的操作有：

- mkdir() - 创建一个新目录
- creat() - 创建一个新的（空）文件
- open(), write(), close() - 向文件追加一个数据块
- link()   - 创建一个文件的硬链接
- unlink() - 取消文件的链接（如果链接计数 linkcnt==0 则删除它）

要理解本作业如何运作，你必须首先理解此文件系统的磁盘状态是如何表示的。文件系统的状态通过打印四个不同数据结构的内容来展示：

- inode 位图：指示哪些 inode 已分配
- inodes：inode 表及其内容
- 数据位图：指示哪些数据块已分配
- 数据：指示数据块的内容

位图应该相当容易理解，1 表示相应的 inode 或数据块已分配，0 表示该 inode 或数据块空闲。

每个 inode 有三个字段：第一个字段指示文件的类型（例如，f 表示常规文件，d 表示目录）；第二个指示哪个数据块属于一个文件（这里，文件只能是空的，这种情况数据块地址设置为 -1，或者大小为一块，这种情况有一个非负地址）；第三个显示文件或目录的引用计数。例如，以下 inode 是一个常规文件，它是空的（地址字段设为 -1），在文件系统中只有一个链接：

```
[f a:-1 r:1]
```

如果同一个文件分配了一个数据块（比如块 10），它将显示如下：

```
[f a:10 r:1]
```

如果有人随后创建了一个指向此 inode 的硬链接，它将变为：

```
[f a:10 r:2]
```

最后，数据块可以保存用户数据或目录数据。如果填充了目录数据，块内的每个条目形式为 (name, inumber)，其中 "name" 是文件或目录的名称，"inumber" 是文件的 inode 编号。因此，假设根 inode 是 0，一个空的根目录看起来像这样：

```
[(.,0) (..,0)]
```

如果我们在根目录添加一个文件 "f"，它被分配了 inode 编号 1，则根目录的内容将变为：

```
[(.,0) (..,0) (f,1)]
```

如果一个数据块包含用户数据，它只显示块内的单个字符，例如 "h"。如果它是空的且未分配，则只显示一对空括号 ([])。

因此，整个文件系统描绘如下：

```
inode bitmap 11110000
inodes       [d a:0 r:3] [f a:1 r:1] [f a:-1 r:1] [d a:2 r:2] [] ...
data bitmap  11100000
data         [(.,0) (..,0) (y,1) (z,2) (f,3)] [u] [(.,3) (..,0)] [] ...
```

这个文件系统有 8 个 inode 和 8 个数据块。根目录包含三个条目（除了 "." 和 ".."），指向 "y"、"z" 和 "f"。通过查找 inode 1，我们可以看到 "y" 是一个常规文件（类型 f），分配了一个数据块（地址 1）。在那个数据块 1 中是文件 "y" 的内容：即 "u"。我们还可以看到 "z" 是一个空的常规文件（地址字段设为 -1），"f"（inode 编号 3）是一个目录，也是空的。你还可以从位图中看到前四个 inode 位图条目被标记为已分配，以及前三个数据位图条目。

模拟器可以使用以下标志运行：

```
prompt> vsfs.py -h
Usage: vsfs.py [options]

Options:
  -h, --help            显示此帮助信息并退出
  -s SEED, --seed=SEED  随机种子
  -i NUMINODES, --numInodes=NUMINODES
                        文件系统中的 inode 数量
  -d NUMDATA, --numData=NUMDATA
                        文件系统中的数据块数量
  -n NUMREQUESTS, --numRequests=NUMREQUESTS
                        要模拟的请求数量
  -r, --reverse         打印操作而不是状态
  -p, --printFinal      打印最终的文件/目录集
  -c, --compute         为我计算答案
```

典型用法是简单地指定一个随机种子（以生成不同的问题）和要模拟的请求数量。在此默认模式下，模拟器打印出文件系统在每个步骤的状态，并询问你必须执行了哪个操作才能使文件系统从一个状态转变到另一个状态。例如：

在这种模式下运行时，模拟器只显示一系列状态，并询问是什么操作导致了这些转变的发生。使用 "-c" 标志运行会向我们展示答案。具体来说，文件 "/y" 被创建，向它追加了一个数据块，从 "/m" 到 "/y" 创建了一个硬链接，通过调用 unlink 删除了 "/m"，创建了文件 "/z"，并创建了目录 "/f"：

```
prompt> vsfs.py -n 6 -s 16 -c

```

其他几个标志控制模拟的各个方面，包括 inode 的数量 ("-i")、数据块的数量 ("-d") 以及是否打印文件系统中所有目录和文件的最终列表 ("-p")。
