# 概述

在本作业中，你将通过一个简单的程序（位于 `mem.c`文件中）来研究交换（swap）的性能。这个程序非常简单：它只是分配一个特定大小的整数数组，然后重复循环遍历这个数组，每次递增数组中的每个值。

输入 `make`来构建它（查看 `Makefile`文件以了解构建的细节）。

然后，输入 `./mem`后跟一个数字来运行它。这个数字是数组的大小（以 MB 为单位）。因此，要运行一个小的数组（大小为 1 MB）：

```
提示符> ./mem 1
```

要运行一个大的数组（大小为 1 GB）：

```
提示符> ./mem 1024
```

该程序打印出每次循环所花费的时间以及带宽（以 MB/s 为单位）。带宽特别值得关注，因为它能让你了解你所使用的系统移动数据的速度；在现代系统上，这个数值很可能在 GB/s 级别。

以下是一个典型运行的输出示例：

```
提示符> ./mem 1000
正在分配 1048576000 字节 (1000.00 MB)
  数组中的整数数量: 262144000
循环 0 耗时 448.11 毫秒 (带宽: 2231.61 MB/s)
循环 1 耗时 345.38 毫秒 (带宽: 2895.38 MB/s)
循环 2 耗时 345.18 毫秒 (带宽: 2897.07 MB/s)
循环 3 耗时 345.23 毫秒 (带宽: 2896.61 MB/s)
^C
提示符>
```

程序首先告诉你它分配了多少内存（以字节、MB 和整数个数表示），然后开始循环遍历数组。第一次循环（在上面的例子中）耗时 448 毫秒；因为程序在不到半秒内访问了 1000 MB 数据，所以计算出的带宽（毫不意外地）略高于 2000 MB/s。

程序会继续对循环 1、2 等重复执行相同的操作。

重要：要停止程序，你必须终止它。在 Linux（以及所有基于 Unix 的系统）上，这可以通过键入 control-C (^C) 来完成，如上所示。

请注意，当你使用小的数组大小时，并非每次循环的性能数据都会被打印出来。例如：

```
提示符>  ./mem 1
正在分配 1048576 字节 (1.00 MB)
  数组中的整数数量: 262144
循环 0 耗时 0.71 毫秒 (带宽: 1414.61 MB/s)
循环 607 耗时 0.33 毫秒 (带宽: 3039.35 MB/s)
循环 1215 耗时 0.33 毫秒 (带宽: 3030.57 MB/s)
循环 1823 耗时 0.33 毫秒 (带宽: 3039.35 MB/s)
^C
提示符>
```

在这种情况下，程序只打印部分输出样本，以免屏幕输出过多。

代码本身很容易理解。第一个重要的部分是内存分配：

```
// 主要的内存分配在这里发生
    int *x = malloc(size_in_bytes);
```

然后，主循环开始：

```
while (1) {
	x[i++] += 1; // 循环的主要工作在此完成。
```

其余部分只是计时和打印信息。详情请见 `mem.c`文件。

本作业的许多内容围绕使用工具 `vmstat`来监视系统发生的情况。请阅读 vmstat 手册页（输入 `man vmstat`）以了解其工作原理以及输出中每一列的含义。
