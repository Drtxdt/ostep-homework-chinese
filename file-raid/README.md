# 概述

本节介绍 `raid.py`，一个简单的 RAID 模拟器，你可以用它来巩固对 RAID 系统工作原理的理解。它有许多选项，如下所示：

```
prompt> ./raid.py -h
用法: raid.py [选项]

选项:
  -h, --help            显示此帮助信息并退出
  -s SEED, --seed=SEED  随机种子
  -D NUMDISKS, --numDisks=NUMDISKS
                        RAID 中的磁盘数量
  -C CHUNKSIZE, --chunkSize=CHUNKSIZE
                        RAID 的块大小
  -n NUMREQUESTS, --numRequests=NUMREQUESTS
                        要模拟的请求数量
  -S SIZE, --reqSize=SIZE
                        请求的大小
  -W WORKLOAD, --workload=WORKLOAD
                        工作负载，"rand" 或 "seq"
  -w WRITEFRAC, --writeFrac=WRITEFRAC
                        写入比例 (100->全部写入, 0->全部读取)
  -R RANGE, --randRange=RANGE
                        请求的范围 (使用 "rand" 工作负载时)
  -L LEVEL, --level=LEVEL
                        RAID 级别 (0, 1, 4, 5)
  -5 RAID5TYPE, --raid5=RAID5TYPE
                        RAID-5 左对称 "LS" 或左非对称 "LA"
  -r, --reverse         显示物理操作而不是逻辑操作
  -t, --timing          使用计时模式，而不是映射模式
  -c, --compute         为我计算答案
```

在基本模式下，你可以用它来理解不同 RAID 级别如何将逻辑块映射到底层磁盘和偏移量。例如，假设我们希望查看一个具有四个磁盘的简单条带化 RAID (RAID-0) 如何进行这种映射。

```
prompt> ./raid.py -n 5 -L 0 -R 20
...
逻辑读取 地址:16 大小:4096
  物理读取/写入?

逻辑读取 地址:8 大小:4096
  物理读取/写入?

逻辑读取 地址:10 大小:4096
  物理读取/写入?

逻辑读取 地址:15 大小:4096
  物理读取/写入?

逻辑读取 地址:9 大小:4096
  物理读取/写入?
```

在这个例子中，我们模拟五个请求 (-n 5)，指定 RAID 级别零 (-L 0)，并将随机请求的范围限制在 RAID 的前二十个块 (-R 20)。结果是对 RAID 前二十个块的一系列随机读取；然后模拟器要求你猜测每个逻辑读取请求需要访问哪些底层磁盘/偏移量。

在这种情况下，计算答案很容易：在 RAID-0 中，回想一下，服务请求的底层磁盘和偏移量是通过取模运算计算的：

```
磁盘   = 地址 % 磁盘数量
偏移量 = 地址 / 磁盘数量
```

因此，第一个对地址 16 的请求应该由磁盘 0 在偏移量 4 处服务。依此类推。你可以像往常一样，通过使用方便的 "-c" 标志来计算结果，从而看到答案（在你计算了它们之后！）。

```
prompt> ./raid.py -R 20 -n 5 -L 0 -c
...
逻辑读取 地址:16 大小:4096
  读取  [磁盘 0, 偏移 4]   

逻辑读取 地址:8 大小:4096
  读取  [磁盘 0, 偏移 2]   

逻辑读取 地址:10 大小:4096
  读取  [磁盘 2, 偏移 2]   

逻辑读取 地址:15 大小:4096
  读取  [磁盘 3, 偏移 3]   

逻辑读取 地址:9 大小:4096
  读取  [磁盘 1, 偏移 2]
```

因为我们喜欢找点乐子，你也可以用 "-r" 标志反向做这个问题。以这种方式运行模拟器会显示底层的磁盘读取和写入，并要求你反推必须向 RAID 发出了什么逻辑请求：

```
prompt> ./raid.py -R 20 -n 5 -L 0 -r
...
逻辑操作是？
  读取  [磁盘 0, 偏移 4]   

逻辑操作是？
  读取  [磁盘 0, 偏移 2]   

逻辑操作是？
  读取  [磁盘 2, 偏移 2]   

逻辑操作是？
  读取  [磁盘 3, 偏移 3]   

逻辑操作是？
  读取  [磁盘 1, 偏移 2]
```

你可以再次使用 -c 来显示答案。为了获得更多变化，可以给出不同的随机种子 (-s)。

通过检查不同的 RAID 级别可以获得更多变化。在模拟器中，支持 RAID-0（块条带化）、RAID-1（镜像）、RAID-4（块条带化加单奇偶校验盘）和 RAID-5（带轮转奇偶校验的块条带化）。

在下一个例子中，我们展示如何在镜像模式下运行模拟器。为了节省空间，我们直接显示答案：

```
prompt> ./raid.py -R 20 -n 5 -L 1 -c
...
逻辑读取 地址:16 大小:4096
  读取  [磁盘 0, 偏移 8]   

逻辑读取 地址:8 大小:4096
  读取  [磁盘 0, 偏移 4]   

逻辑读取 地址:10 大小:4096
  读取  [磁盘 1, 偏移 5]   

逻辑读取 地址:15 大小:4096
  读取  [磁盘 3, 偏移 7]   

逻辑读取 地址:9 大小:4096
  读取  [磁盘 2, 偏移 4]
```

你可能会注意到这个例子的一些事情。首先，镜像 RAID-1 假设了条带化布局（有些人可能称之为 RAID-10，或镜像的条带化），其中逻辑块 0 映射到磁盘 0 和 1 的第 0 块，逻辑块 1 映射到磁盘 2 和 3 的第 0 块，依此类推（在这个四磁盘的例子中）。其次，当从镜像 RAID 系统读取单个块时，RAID 可以选择读取两个块中的哪一个。在这个模拟器中，我们使用一种相对简单的方法：对于偶数编号的逻辑块，RAID 选择对中的偶数编号磁盘；奇数磁盘用于奇数编号的逻辑块。这样做是为了让你更容易猜测每次运行的结果（而不是，例如，随机选择）。

我们还可以用 -w 标志探索写入行为（而不仅仅是读取），该标志指定工作负载的"写入比例"，即，写入请求的比例。默认情况下，它设置为零，因此到目前为止的例子是 100% 读取。让我们看看当引入一些写入时，我们的镜像 RAID 会发生什么：

```
prompt> ./raid.py -R 20 -n 5 -L 1 -w 100 -c
...
逻辑写入 地址:16 大小:4096
  写入  [磁盘 0, 偏移 8]     写入  [磁盘 1, 偏移 8]   

逻辑写入 地址:8 大小:4096
  写入  [磁盘 0, 偏移 4]     写入  [磁盘 1, 偏移 4]   

逻辑写入 地址:10 大小:4096
  写入  [磁盘 0, 偏移 5]     写入  [磁盘 1, 偏移 5]   

逻辑写入 地址:15 大小:4096
  写入  [磁盘 2, 偏移 7]     写入  [磁盘 3, 偏移 7]   

逻辑写入 地址:9 大小:4096
  写入  [磁盘 2, 偏移 4]     写入  [磁盘 3, 偏移 4]
```

对于写入，RAID 当然必须更新两个磁盘，因此会发出两次写入，而不是只生成一个底层磁盘操作。正如你可能会猜到的，RAID-4 和 RAID-5 会发生更有趣的事情；我们将在下面的问题中把这些的探索留给你。

其余选项通过帮助标志发现。它们是：

```
选项:
  -h, --help            显示此帮助信息并退出
  -s SEED, --seed=SEED  随机种子
  -D NUMDISKS, --numDisks=NUMDISKS
                        RAID 中的磁盘数量
  -C CHUNKSIZE, --chunkSize=CHUNKSIZE
                        RAID 的块大小
  -n NUMREQUESTS, --numRequests=NUMREQUESTS
                        要模拟的请求数量
  -S SIZE, --reqSize=SIZE
                        请求的大小
  -W WORKLOAD, --workload=WORKLOAD
                        工作负载，"rand" 或 "seq"
  -w WRITEFRAC, --writeFrac=WRITEFRAC
                        写入比例 (100->全部写入, 0->全部读取)
  -R RANGE, --randRange=RANGE
                        请求的范围 (使用 "rand" 工作负载时)
  -L LEVEL, --level=LEVEL
                        RAID 级别 (0, 1, 4, 5)
  -5 RAID5TYPE, --raid5=RAID5TYPE
                        RAID-5 左对称 "LS" 或左非对称 "LA"
  -r, --reverse         显示物理操作而不是逻辑操作
  -t, --timing          使用计时模式，而不是映射模式
  -c, --compute         为我计算答案
```

-C 标志允许你设置 RAID 的块大小，而不是使用默认的每个块 4 KB。每个请求的大小可以类似地用 -S 标志调整。默认工作负载访问随机块；使用 -W sequential 来探索顺序访问的行为。对于 RAID-5，有两种不同的布局方案可用，左对称和左非对称；使用 -5 LS 或 -5 LA 与 RAID-5 (-L 5) 一起尝试。

最后，在计时模式 (-t) 下，模拟器使用一个极其简单的磁盘模型来估计一组请求所需的时间，而不是仅仅关注映射。在这种模式下，随机请求需要 10 毫秒，而顺序请求需要 0.1 毫秒。假定磁盘每个磁道的块数很少 (100)，磁道数同样很少 (100)。因此，你可以使用模拟器来估计 RAID 在某些不同工作负载下的性能。
