# 概述

这个程序 `malloc.py`允许你了解一个简单内存分配器的工作原理。以下是你可以使用的选项：

```
-h, --help            显示此帮助信息并退出
  -s SEED, --seed=SEED  随机种子
  -S HEAPSIZE, --size=HEAPSIZE
                        堆的大小
  -b BASEADDR, --baseAddr=BASEADDR
                        堆的基地址
  -H HEADERSIZE, --headerSize=HEADERSIZE
                        每个分配块头部的大小
  -a ALIGNMENT, --alignment=ALIGNMENT
                        将分配的单元对齐到指定大小；-1 表示不对齐
  -p POLICY, --policy=POLICY
                        空闲列表搜索策略 (BEST, WORST, FIRST)
  -l ORDER, --listOrder=ORDER
                        空闲列表排序方式 (ADDRSORT, SIZESORT+, SIZESORT-, INSERT-FRONT, INSERT-BACK)
  -C, --coalesce        合并空闲列表？
  -n OPSNUM, --numOps=OPSNUM
                        要生成的随机操作数量
  -r OPSRANGE, --range=OPSRANGE
                        最大分配大小
  -P OPSPALLOC, --percentAlloc=OPSPALLOC
                        分配操作所占的百分比
  -A OPSLIST, --allocList=OPSLIST
                        代替随机生成，指定操作列表 (+10,-0, 等)
  -c, --compute         为我计算答案
```

使用它的一种方法是让程序生成一些随机的分配/释放操作，然后让你尝试计算出空闲列表的样子，以及每个操作的成功或失败。

下面是一个简单的例子：

```
提示符> ./malloc.py -S 100 -b 1000 -H 4 -a 4 -l ADDRSORT -p BEST -n 5

ptr[0] = Alloc(3)  返回 ?
列表?

Free(ptr[0]) 返回 ?
列表?

ptr[1] = Alloc(5)  返回 ?
列表?

Free(ptr[1]) 返回 ?
列表?

ptr[2] = Alloc(8)  返回 ?
列表?
```

在这个例子中，我们指定了一个大小为 100 字节的堆 (-S 100)，起始地址为 1000 (-b 1000)。我们指定每个分配块有额外的 4 字节头部 (-H 4)，并确保每个分配的空间向上取整到最近的 4 字节空闲块大小 (-a 4)。我们指定空闲列表按地址（递增）排序。最后，我们指定“最佳匹配”空闲列表搜索策略 (-p BEST)，并要求生成 5 个随机操作 (-n 5)。运行此命令的结果如上所示；你的任务是找出每个分配/释放操作返回什么，以及每次操作后空闲列表的状态。

这里我们通过使用 -c 选项来查看结果。

```
提示符> ./malloc.py -S 100 -b 1000 -H 4 -a 4 -l ADDRSORT -p BEST -n 5 -c

ptr[0] = Alloc(3)  返回 1004 (搜索了 1 个元素)
空闲列表 [ 大小 1 ]:  [ 地址:1008 大小:92 ]

Free(ptr[0]) 返回 0
空闲列表 [ 大小 2 ]:  [ 地址:1000 大小:8 ] [ 地址:1008 大小:92 ]

ptr[1] = Alloc(5)  返回 1012 (搜索了 2 个元素)
空闲列表 [ 大小 2 ]:  [ 地址:1000 大小:8 ] [ 地址:1020 大小:80 ]

Free(ptr[1]) 返回 0
空闲列表 [ 大小 3 ]:  [ 地址:1000 大小:8 ] [ 地址:1008 大小:12 ] [ 地址:1020 大小:80 ]

ptr[2] = Alloc(8)  返回 1012 (搜索了 3 个元素)
空闲列表 [ 大小 2 ]:  [ 地址:1000 大小:8 ] [ 地址:1020 大小:80 ]
```

如你所见，第一个分配操作返回以下信息：

```
ptr[0] = Alloc(3)  返回 1004 (搜索了 1 个元素)
空闲列表 [ 大小 1 ]:  [ 地址:1008 大小:92 ]
```

由于初始的空闲列表状态只是一个大的元素，很容易猜到 Alloc(3) 请求会成功。此外，它将只返回第一个内存块，并将剩余部分作为空闲列表。返回的指针将紧跟在头部之后（地址：1004），并且分配的空间向上取整到 4 字节，使得空闲列表从地址 1008 开始，大小为 92 字节。

下一个操作是释放 "ptr[0]"，它存储了前一个分配请求的结果。正如你所料，这次释放会成功（因此返回 "0"），而现在的空闲列表看起来稍微复杂一些：

```
Free(ptr[0]) 返回 0
空闲列表 [ 大小 2 ]:  [ 地址:1000 大小:8 ] [ 地址:1008 大小:92 ]
```

事实上，因为我们**没有**合并空闲列表，所以现在它上面有两个元素，第一个是 8 字节大小，包含刚释放的空间，第二个是 92 字节的块。

我们确实可以通过 -C 标志打开合并功能，结果如下：

```
提示符> ./malloc.py -S 100 -b 1000 -H 4 -a 4 -l ADDRSORT -p BEST -n 5 -c -C
ptr[0] = Alloc(3)  返回 1004 (搜索了 1 个元素)
空闲列表 [ 大小 1 ]:  [ 地址:1008 大小:92 ]

Free(ptr[0]) 返回 0
空闲列表 [ 大小 1 ]:  [ 地址:1000 大小:100 ]

ptr[1] = Alloc(5)  返回 1004 (搜索了 1 个元素)
空闲列表 [ 大小 1 ]:  [ 地址:1012 大小:88 ]

Free(ptr[1]) 返回 0
空闲列表 [ 大小 1 ]:  [ 地址:1000 大小:100 ]

ptr[2] = Alloc(8)  返回 1004 (搜索了 1 个元素)
空闲列表 [ 大小 1 ]:  [ 地址:1012 大小:88 ]
```

你可以看到，当执行释放操作时，空闲列表会按预期进行合并。

还有一些其他有趣的选项值得探索：

- `-p BEST`或 `-p WORST`或 `-p FIRST`：此选项允许你使用这三种不同的策略在分配请求期间寻找一块内存。
- `-l ADDRSORT`或 `-l SIZESORT+`或 `-l SIZESORT-`或 `-l INSERT-FRONT`或 `-l INSERT-BACK`：此选项允许你按照特定顺序维护空闲列表，例如按空闲块的地址排序、按空闲块的大小排序（递增用 +，递减用 -），或者简单地将释放的块放回空闲列表的前端 (INSERT-FRONT) 或后端 (INSERT-BACK)。
- `-A 操作列表`：此选项允许你指定一个确切的操作序列，而不是随机生成的。例如，使用参数 "-A +10,+10,+10,-0,-2" 运行将分配三个大小为 10 字节（加上头部）的块，然后释放第一个 ("-0")，再释放第三个 ("-2")。那时空闲列表会是什么样子呢？

这些是基本知识。使用书本章节中的问题来进一步探索，或者自己创建新的有趣问题，以便更好地理解分配器是如何工作的。
