# 概述

此程序 scheduler.py 可让你观察不同的调度器在调度指标下的表现，例如响应时间、周转时间和总等待时间。程序"实现"了三种调度器：FIFO、SJF 和 RR。

运行程序分为两个步骤。

首先，在不使用 -c 标志的情况下运行：这会展示需要解决的问题而不给出答案。例如，如果你想使用 FIFO 策略计算三个作业的响应时间、周转时间和等待时间，运行：

```
提示符> ./scheduler.py -p FIFO -j 3 -s 100
```

如果上述命令无效，请尝试：

```
提示符> python ./scheduler.py -p FIFO -j 3 -s 100
```

这指定了使用 FIFO 策略、三个作业，重要的是，随机种子指定为 100。如果你想查看此确切问题的解决方案，必须再次指定这个完全相同的随机种子。让我们运行看看会发生什么。你应该看到：

```
提示符> ./scheduler.py -p FIFO -j 3 -s 100
参数 调度策略 FIFO
参数 作业数量 3
参数 最大长度 10
参数 随机种子 100

以下是作业列表，包含每个作业的运行时间：
  作业 0 (长度 = 1)
  作业 1 (长度 = 4)
  作业 2 (长度 = 7)
```

计算每个作业的周转时间、响应时间和等待时间。完成后，再次运行此程序，使用相同的参数，但加上 -c 标志，这将为你提供答案。你可以使用 -s <某个数字> 或你自己的作业列表（例如 -l 10,15,20）为自己生成不同的问题。

如本例所示，生成了三个作业：长度为 1 的作业 0，长度为 4 的作业 1，以及长度为 7 的作业 2。如程序所述，你现在可以用此计算一些统计量，看看是否掌握了基本概念。

完成后，你可以使用相同的程序来"解答"此问题，并检查你的计算是否正确。为此，使用 "-c" 标志。输出：

```
提示符> ./scheduler.py -p FIFO -j 3 -s 100 -c
参数 调度策略 FIFO
参数 作业数量 3
参数 最大长度 10
参数 随机种子 100

以下是作业列表，包含每个作业的运行时间：
  作业 0 (长度 = 1)
  作业 1 (长度 = 4)
  作业 2 (长度 = 7)

** 解答 **

执行轨迹：
  [时间 0] 运行作业 0 1.00 秒 (完成)
  [时间 1] 运行作业 1 4.00 秒 (完成)
  [时间 5] 运行作业 2 7.00 秒 (完成)

最终统计：
  作业 0 -- 响应: 0.00  周转 1.00  等待 0.00
  作业 1 -- 响应: 1.00  周转 5.00  等待 1.00
  作业 2 -- 响应: 5.00  周转 12.00  等待 5.00

  平均 -- 响应: 2.00  周转 6.00  等待 2.00
```

如图中所示，-c 标志显示了发生的情况。作业 0 首先运行 1 秒，作业 1 其次运行 4 秒，然后作业 2 运行 7 秒。这并不难；毕竟是 FIFO！执行轨迹显示了这些结果。

最终统计也很有用：它们计算了"响应时间"（作业到达后首次运行前的等待时间）、"周转时间"（从首次到达开始到作业完成所需的时间）和总"等待时间"（就绪但未运行的任何时间）。统计信息按作业显示，然后是所有作业的平均值。当然，你应该在运行 "-c" 标志之前就计算好这些！

如果你想尝试同类但输入不同的问题，可以更改作业数量或随机种子，或两者都改。不同的随机种子基本上为你提供了一种为自己生成无限多不同问题的方法，而 "-c" 标志可以让你检查自己的作业。不断练习，直到你感觉真正理解了这些概念。

另一个有用的标志是 "-l"（小写字母 L），它可以让你指定希望调度的确切作业。例如，如果你想了解 SJF 策略在作业长度为 5、10 和 15 的三个作业下的表现，可以运行：

```
提示符> ./scheduler.py -p SJF -l 5,10,15
参数 调度策略 SJF
参数 作业列表 5,10,15

以下是作业列表，包含每个作业的运行时间：
  作业 0 (长度 = 5.0)
  作业 1 (长度 = 10.0)
  作业 2 (长度 = 15.0)
...
```

然后你可以再次使用 -c 来解答它。请注意，当你指定确切作业时，无需指定随机种子或作业数量：作业长度取自你以逗号分隔的列表。

当然，当你使用 SJF（最短作业优先）或甚至 RR（轮转）调度器时，会发生更有趣的事情。试试看吧！

你随时可以运行

```
提示符> ./scheduler.py -h
```

来获取完整的标志和选项列表（包括为 RR 调度器设置时间量子的选项）。
