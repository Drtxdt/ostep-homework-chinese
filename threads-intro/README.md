# 概述

欢迎使用这个模拟器！本模拟器的理念是通过观察线程如何交错执行来熟悉线程；模拟器 `x86.py`将帮助你获得这种理解。

该模拟器模拟多个线程对简短汇编序列的执行。注意，模拟器**不显示**操作系统代码（例如，执行上下文切换的代码）的运行；因此，你看到的所有内容都是用户代码的交错执行。

运行的汇编代码基于 x86，但有所简化。在这个指令集中，有四个通用寄存器（%ax, %bx, %cx, %dx），一个程序计数器 (PC)，以及一组足以满足我们目的的少量指令。

以下是我们将要运行的示例代码片段：

```
.main
mov 2000, %ax   # 获取地址处的值
add $1, %ax     # 递增它
mov %ax, 2000   # 存回
halt
```

这段代码很容易理解。第一条指令是 x86 的 "mov"，它简单地将地址 2000 指定的值加载到寄存器 %ax 中。在这个 x86 子集中，地址可以采用以下一些形式：

- `2000`: 数字 (2000) 就是地址
- `(%cx)`: 寄存器中的内容（括号内）构成地址
- `1000(%dx)`: 数字 + 寄存器的内容构成地址
- `10(%ax,%bx)`: 数字 + 寄存器1 + 寄存器2 构成地址

要存储一个值，使用相同的 `mov`指令，但这次参数顺序相反，例如：

```
mov %ax, 2000
```

上面序列中的 `add`指令应该很清楚：它将一个立即数（由 `$1`指定）加到第二个参数指定的寄存器中（即 `%ax = %ax + 1`）。

因此，我们现在可以理解上面的代码序列：它加载地址 2000 处的值，将其加 1，然后将该值存回地址 2000。

伪造的 `halt`指令只是停止运行此线程。

让我们运行模拟器，看看这一切是如何工作的！假设上面的代码序列在文件 `simple-race.s`中。

```
prompt> ./x86.py -p simple-race.s -t 1

       线程 0
1000 mov 2000, %ax
1001 add $1, %ax
1002 mov %ax, 2000
1003 halt

prompt>
```

这里使用的参数指定了程序 (`-p`)、线程数量 (`-t 1`) 和中断间隔，后者是调度器被唤醒并运行以切换到另一个任务的频率。由于此示例中只有一个线程，此间隔无关紧要。

输出很容易阅读：模拟器打印程序计数器（此处显示为 1000 到 1003）和要执行的指令。注意，我们（不现实地）假设所有指令在内存中只占用一个字节；在 x86 中，指令是可变大小的，会占用一个到几个字节。

我们可以使用更详细的跟踪来更好地了解机器状态在执行过程中的变化：

```
prompt> ./x86.py -p simple-race.s -t 1 -M 2000 -R ax,bx

 2000      ax    bx          线程 0
    ?       ?     ?
    ?       ?     ?   1000 mov 2000, %ax
    ?       ?     ?   1001 add $1, %ax
    ?       ?     ?   1002 mov %ax, 2000
    ?       ?     ?   1003 halt

哎呀！忘了 -c 标志（它会为你计算答案）。

prompt> ./x86.py -p simple-race.s -t 1 -M 2000 -R ax,bx -c

 2000      ax    bx          线程 0
    0       0     0
    0       0     0   1000 mov 2000, %ax
    0       1     0   1001 add $1, %ax
    1       1     0   1002 mov %ax, 2000
    1       1     0   1003 halt
```

通过使用 `-M`标志，我们可以跟踪内存位置（逗号分隔的列表允许你跟踪多个位置，例如 2000,3000）；通过使用 `-R`标志，我们可以跟踪特定寄存器内的值。

左侧的值显示在右侧指令执行**之后**的内存/寄存器内容。例如，在 `add`指令之后，你可以看到 %ax 已递增到值 1；在第二条 `mov`指令（PC=1002 处）之后，你可以看到地址 2000 处的内存内容现在也已递增。

你还需要了解一些更多指令，所以现在让我们来介绍它们。以下是一个循环的代码片段：

```
.main
.top
sub  $1,%dx
test $0,%dx     
jgte .top         
halt
```

这里引入了一些新内容。首先是 `test`指令。该指令接受两个参数并比较它们；然后设置隐式的"条件码"（有点像 1 位寄存器），后续指令可以基于此进行操作。

在这种情况下，另一个新指令是 `jump`指令（这里是 `jgte`，代表"大于或等于时跳转"）。如果 `test`中的第二个值大于或等于第一个值，则该指令执行跳转。

最后一点：为了让这段代码真正工作，`dx`必须初始化为 1 或更大。

因此，我们像这样运行程序：

```
prompt> ./x86.py -p loop.s -t 1 -a dx=3 -R dx -C -c

   dx   >= >  <= <  != ==        线程 0
    3   0  0  0  0  0  0
    2   0  0  0  0  0  0  1000 sub  $1,%dx
    2   1  1  0  0  1  0  1001 test $0,%dx
    2   1  1  0  0  1  0  1002 jgte .top
    1   1  1  0  0  1  0  1000 sub  $1,%dx
    1   1  1  0  0  1  0  1001 test $0,%dx
    1   1  1  0  0  1  0  1002 jgte .top
    0   1  1  0  0  1  0  1000 sub  $1,%dx
    0   1  0  1  0  0  1  1001 test $0,%dx
    0   1  0  1  0  0  1  1002 jgte .top
   -1   1  0  1  0  0  1  1000 sub  $1,%dx
   -1   0  0  1  1  1  0  1001 test $0,%dx
   -1   0  0  1  1  1  0  1002 jgte .top
   -1   0  0  1  1  1  0  1003 halt
```

`-R dx`标志跟踪 %dx 的值；`-C`标志跟踪由 `test`指令设置的条件码的值。最后，`-a dx=3`标志将 `%dx`寄存器初始化为值 3。

从跟踪中可以看到，`sub`指令逐渐降低 %dx 的值。前几次调用 `test`时，只有 ">="、">" 和 "!=" 条件被设置。然而，跟踪中的最后一次 `test`发现 %dx 等于 0，因此随后的跳转**不**发生，程序最终停止。

现在，我们终于到了一个更有趣的情况，即具有多个线程的竞态条件。让我们先看一下代码：

```
.main
.top
# 临界区
mov 2000, %ax       # 获取地址处的值
add $1, %ax         # 递增它
mov %ax, 2000       # 存回

# 检查是否仍在循环
sub  $1, %bx
test $0, %bx
jgt .top

halt
```

这段代码有一个临界区，它加载一个变量的值（在地址 2000 处），然后将该值加 1，再存回。

之后的代码只是递减一个循环计数器（在 %bx 中），测试它是否大于或等于零，如果是，则跳回顶部的临界区再次执行。

```
prompt> ./x86.py -p looping-race-nolock.s -t 2 -a bx=1 -M 2000 -c

 2000      bx          线程 0                线程 1
    0       1
    0       1   1000 mov 2000, %ax
    0       1   1001 add $1, %ax
    1       1   1002 mov %ax, 2000
    1       0   1003 sub  $1, %bx
    1       0   1004 test $0, %bx
    1       0   1005 jgt .top
    1       0   1006 halt
    1       1   ----- 暂停;切换 -----  ----- 暂停;切换 -----
    1       1                            1000 mov 2000, %ax
    1       1                            1001 add $1, %ax
    2       1                            1002 mov %ax, 2000
    2       0                            1003 sub  $1, %bx
    2       0                            1004 test $0, %bx
    2       0                            1005 jgt .top
    2       0                            1006 halt
```

在这里，你可以看到每个线程运行了一次，并且每个线程都更新了一次地址 2000 处的共享变量，因此最终结果为 2。

每当一个线程暂停且必须运行另一个线程时，会插入 `暂停;切换`行。

最后一个示例：运行与上面相同的代码，但使用较小的中断频率。结果如下：

```
prompt> ./x86.py -p looping-race-nolock.s -t 2 -a bx=1 -M 2000 -i 2

 2000          线程 0                线程 1
    ?
    ?   1000 mov 2000, %ax
    ?   1001 add $1, %ax
    ?   ------ 中断 ------  ------ 中断 ------
    ?                            1000 mov 2000, %ax
    ?                            1001 add $1, %ax
    ?   ------ 中断 ------  ------ 中断 ------
    ?   1002 mov %ax, 2000
    ?   1003 sub  $1, %bx
    ?   ------ 中断 ------  ------ 中断 ------
    ?                            1002 mov %ax, 2000
    ?                            1003 sub  $1, %bx
    ?   ------ 中断 ------  ------ 中断 ------
    ?   1004 test $0, %bx
    ?   1005 jgt .top
    ?   ------ 中断 ------  ------ 中断 ------
    ?                            1004 test $0, %bx
    ?                            1005 jgt .top
    ?   ------ 中断 ------  ------ 中断 ------
    ?   1006 halt
    ?   ----- 暂停;切换 -----  ----- 暂停;切换 -----
    ?                            1006 halt
```

如你所见，每个线程每 2 条指令被中断一次，这是通过 `-i 2`标志指定的。这次运行中 memory[2000] 的值是多少？它应该是多少？

现在让我们提供更多关于这个程序可以模拟什么的信息。完整的寄存器集：%ax, %bx, %cx, %dx 和 PC。在这个版本中，不支持"栈"，也没有调用和返回指令。

模拟的完整指令集如下：

```
mov immediate, register     # 将立即数移动到寄存器
mov memory, register        # 从内存加载到寄存器
mov register, register      # 将值从一个寄存器移动到另一个寄存器
mov register, memory        # 将寄存器内容存储到内存
mov immediate, memory       # 将立即数存储到内存

add immediate, register     # 寄存器 = 寄存器 + 立即数
add register1, register2    # 寄存器2 = 寄存器2 + 寄存器1
sub immediate, register     # 寄存器 = 寄存器 - 立即数
sub register1, register2    # 寄存器2 = 寄存器2 - 寄存器1

test immediate, register    # 比较立即数和寄存器（设置条件码）
test register, immediate    # 同上，但寄存器和立即数
test register, register     # 同上，但寄存器和寄存器

jne                         # 如果测试值不相等则跳转
je                          #             ... 相等
jlt                         # 第二个小于第一个
jlte                        # 小于或等于
jgt                         # 大于
jgte                        # 大于或等于

xchg register, memory       # 原子交换：
                            #   将寄存器的值放入内存
                            #   将内存的旧内容返回到寄存器
                            #   原子地完成这两件事

nop                         # 空操作
```

注意：

- 'immediate' 是 $number 形式
- 'memory' 是 'number' 或 '(reg)' 或 'number(reg)' 或 'number(reg,reg)' 形式（如上所述）
- 'register' 是 %ax, %bx, %cx, %dx 之一

最后，模拟器的完整选项集可以通过 `-h`标志查看：

```
用法: x86.py [选项]

选项:
  -h, --help            显示此帮助信息并退出
  -s SEED, --seed=SEED  随机种子
  -t NUMTHREADS, --threads=NUMTHREADS
                        线程数
  -p PROGFILE, --program=PROGFILE
                        源程序（.s 文件）
  -i INTFREQ, --interrupt=INTFREQ
                        中断频率
  -r, --randints        如果中断是随机的
  -a ARGV, --argv=ARGV  每个线程的逗号分隔参数（例如，ax=1,ax=2 将线程 0 的 ax 寄存器设为 1，线程 1 的 ax 寄存器设为 2）；
                        每个线程通过冒号分隔的列表指定多个寄存器（例如，ax=1:bx=2,cx=3 设置线程 0 的 ax 和 bx，线程 1 的 cx）
  -L LOADADDR, --loadaddr=LOADADDR
                        加载代码的地址
  -m MEMSIZE, --memsize=MEMSIZE
                        地址空间大小（KB）
  -M MEMTRACE, --memtrace=MEMTRACE
                        逗号分隔的要跟踪的地址列表（例如，20000,20001）
  -R REGTRACE, --regtrace=REGTRACE
                        逗号分隔的要跟踪的寄存器列表（例如，ax,bx,cx,dx）
  -C, --cctrace         是否跟踪条件码
  -S, --printstats      打印一些额外统计信息
  -c, --compute         为我计算答案
```

大多数选项是显而易见的。使用 `-r`会开启随机中断（从 1 到由 `-i`指定的 intfreq 之间的随机值），这可以使作业问题更有趣。

- `-L`指定在地址空间中加载代码的位置。
- `-m`指定地址空间的大小（以 KB 为单位）。
- `-S`打印一些额外统计信息
- `-c`并未真正使用（不像书中的大多数模拟器）；使用跟踪或条件码。
