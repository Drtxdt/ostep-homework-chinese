# 概述

这份作业让你尝试在 C 语言中实现一个避免死锁的小型向量对象的多种方法。这个向量对象功能相当有限（例如，它只有 `add()`和 `init()`函数），但主要用于说明避免死锁的不同方法。

你应该关注的一些文件如下。特别是，本作业中的所有变体都会用到它们。

- `common_threads.h`: 对许多不同的 pthread（和其他）库调用的常用包装函数，以确保它们不会静默失败
- `vector-header.h`: 向量例程的一个简单头文件，主要定义了固定的向量大小，以及每个向量使用的结构体 (vector_t)
- `main-header.h`: 每个不同程序共有的多个全局变量
- `main-common.c`: 包含主函数 main() 例程（带有参数解析），它初始化两个向量，启动一些线程来访问它们（通过 worker() 例程），然后等待多次 vector_add() 完成

本作业的变体可以在以下文件中找到。每个文件对名为 `vector_add()`的"向量加法"例程中的并发处理采取了不同的方法；检查这些文件中的代码以了解具体发生了什么。它们都使用上述文件来构成一个完整的可运行程序。

相关文件：

- `vector-deadlock.c`: 此版本以一种特定的顺序（先 dst 后 src）盲目地获取锁。通过这样做，它创建了一个"死锁的诱因"，因为一个线程可能调用 `vector_add(v1, v2)`，而另一个线程同时调用 `vector_add(v2, v1)`。
- `vector-global-order.c`: 此版本的 `vector_add()`基于向量的地址，按照一个全局顺序获取锁。
- `vector-try-wait.c`: 此版本的 `vector_add()`使用 `pthread_mutex_trylock()`来尝试获取锁；当尝试失败时，代码释放它可能持有的任何锁，然后回到顶部重新尝试。
- `vector-avoid-hold-and-wait.c`: 此版本通过在锁获取周围使用单个锁，确保自己不会陷入持有并等待的模式。
- `vector-nolock.c`: 此版本甚至不使用锁；相反，它使用原子 fetch-and-add 操作来实现 `vector_add()`例程。因此，它的语义略有不同。

输入 `make`（并阅读 `Makefile`）来构建五个可执行文件。

```
prompt> make
```

然后，你可以通过简单地输入其名称来运行一个程序：

```
prompt> ./vector-deadlock
```

每个程序接受相同的参数集（详见 main-common.c）：

- `-d`: 此标志启用线程死锁的可能性。当你将 `-d`传递给程序时，每隔一个线程会以不同的顺序调用 `vector_add()`，例如，有两个线程且启用 `-d`时，线程 0 调用 `vector_add(v1, v2)`，线程 1 调用 `vector_add(v2, v1)`。
- `-p`: 此标志为每个线程提供一组不同的向量来调用 add 操作，而不是仅使用两个向量。使用此标志以查看当对同一组向量没有争用时，程序如何运行。
- `-n num_threads`: 创建指定数量的线程；你需要多于一个线程才能产生死锁。
- `-l loops`: 每个线程应调用 add 多少次？
- `-v`: 详细标志：打印更多关于正在发生的情况。
- `-t`: 启用计时并显示整个过程花费的时间。
