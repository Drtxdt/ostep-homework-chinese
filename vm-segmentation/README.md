# 概述

这个程序允许你查看在使用分段（segmentation）的系统中地址转换是如何执行的。该系统使用的分段非常简单：一个地址空间只有*两个*段；此外，进程生成的虚拟地址的最高位决定了该地址属于哪个段：0 表示段 0（例如，代码和堆所在的位置），1 表示段 1（栈所在的位置）。段 0 向正方向增长（朝向更高的地址），而段 1 向负方向增长。

从视觉上看，地址空间如下所示：

```
--------------- 虚拟地址 0
 |    段0      |
 |             |
 |             |
 |-------------|
 |             |
 |             |
 |             |
 |             |
 |  (未分配)    |
 |             |
 |             |
 |             |
 |-------------|
 |             |
 |    段1      |
 |-------------| 虚拟地址最大值（地址空间大小）
```

如你所知，在分段中，每个段都有一对基址/界限寄存器。因此，在这个问题中，有两对基址/界限寄存器。段 0 的基址（base）告诉我们段 0 的*顶部*被放置在物理内存中的哪个位置，界限（limit）告诉我们这个段有多大；段 1 的基址告诉我们段 1 的*底部*被放置在物理内存中的哪个位置，相应的界限也告诉我们这个段有多大（或者它向负方向增长了多少）。

和之前一样，运行程序来测试你对分段的理解分为两个步骤。首先，不使用 "-c" 标志运行，以生成一组转换，并查看你自己是否能正确执行地址转换。然后，完成后，使用 "-c" 标志运行以检查你的答案。

例如，要使用默认标志运行，输入：

```
提示符> ./segmentation.py
```

或

```
提示符> python ./segmentation.py
```

你应该会看到：

```
ARG seed 0
  ARG address space size 1k
  ARG phys mem size 16k
  
  Segment register information:

    Segment 0 base  (grows positive) : 0x00001aea (decimal 6890)
    Segment 0 limit                  : 472

    Segment 1 base  (grows negative) : 0x00001254 (decimal 4692)
    Segment 1 limit                  : 450

  Virtual Address Trace
    VA  0: 0x0000020b (decimal:  523) --> 物理地址 或 段违规？
    VA  1: 0x0000019e (decimal:  414) --> 物理地址 或 段违规？
    VA  2: 0x00000322 (decimal:  802) --> 物理地址 或 段违规？
    VA  3: 0x00000136 (decimal:  310) --> 物理地址 或 段违规？
    VA  4: 0x000001e8 (decimal:  488) --> 物理地址 或 段违规？

  对于每个虚拟地址，要么写下它转换到的物理地址，要么写下它是一个越界地址（分段违规）。
  对于这个问题，你应该假设一个具有两个段的简单地址空间：因此可以使用虚拟地址的最高位来检查
  虚拟地址是在段 0 (最高位=0) 还是段 1 (最高位=1)。注意，给你的基址/界限对根据段的不同
  向不同的方向增长，即段 0 向正方向增长，而段 1 向负方向增长。
```

然后，在你计算出虚拟地址跟踪中的转换后，再次使用 "-c" 标志运行程序。你将看到以下内容（不包括冗余信息）：

```
Virtual Address Trace
    VA  0: 0x0000020b (decimal:  523) --> 分段违规 (SEG1)
    VA  1: 0x0000019e (decimal:  414) --> 在 SEG0 中有效: 0x00001c88 (decimal: 7304)
    VA  2: 0x00000322 (decimal:  802) --> 在 SEG1 中有效: 0x00001176 (decimal: 4470)
    VA  3: 0x00000136 (decimal:  310) --> 在 SEG0 中有效: 0x00001c20 (decimal: 7200)
    VA  4: 0x000001e8 (decimal:  488) --> 分段违规 (SEG0)
```

如你所见，使用 -c 时，程序会为你转换地址，因此你可以检查自己是否理解使用分段的系统如何转换地址。

当然，你可以使用一些参数来给自己出不同的问题。一个特别重要的参数是 -s 或 -seed 参数，它允许你通过传入不同的随机种子来生成不同的问题。当然，在生成问题并解决它时，请确保使用相同的随机种子。

还有一些参数可以让你使用不同大小的地址空间和物理内存。例如，要在一个小型系统中试验分段，你可以输入：

```
提示符> ./segmentation.py -s 100 -a 16 -p 32
ARG seed 0
ARG address space size 16
ARG phys mem size 32
 
Segment register information:

  Segment 0 base  (grows positive) : 0x00000018 (decimal 24)
  Segment 0 limit                  : 4

  Segment 1 base  (grows negative) : 0x00000012 (decimal 18)
  Segment 1 limit                  : 5

Virtual Address Trace
  VA  0: 0x0000000c (decimal:   12) --> 物理地址 或 段违规？
  VA  1: 0x00000008 (decimal:    8) --> 物理地址 或 段违规？
  VA  2: 0x00000001 (decimal:    1) --> 物理地址 或 段违规？
  VA  3: 0x00000007 (decimal:    7) --> 物理地址 或 段违规？
  VA  4: 0x00000000 (decimal:    0) --> 物理地址 或 段违规？
```

这告诉程序为一个放置在 32 字节物理内存中某处的 16 字节地址空间生成虚拟地址。如你所见，生成的虚拟地址很小（12, 8, 1, 7, 和 0）。你还可以看到，程序选择了很小的基址寄存器值和界限值，这是合适的。使用 -c 运行以查看答案。

这个例子也应该向你展示每对基址/界限的确切含义。例如，段 0 的基址设置为物理地址 24（十进制），大小为 4 字节。因此，*虚拟*地址 0、1、2 和 3 在段 0 中并且有效，分别映射到物理地址 24、25、26 和 27。

稍微复杂的是向负方向增长的段 1。在上面的小例子中，段 1 的基址寄存器设置为物理地址 18，大小为 5 字节。这意味着虚拟地址空间的*最后*五个字节，在本例中是 11、12、13、14 和 15，是有效的虚拟地址，并且它们分别映射到物理地址 13、14、15、16 和 17。

如果这说不通，请再读一遍——你必须理解它是如何工作的，才能解决任何这类问题。

请注意，你可以通过在传递给 -a 或 -p 标志的值后加上 "k"、"m" 甚至 "g" 来指定更大的值，分别表示 "千字节"、"兆字节" 和 "千兆字节"。因此，如果你想在一个 32 MB 的物理内存中设置一个 1 MB 的地址空间并进行一些转换，你可以输入：

```
提示符> ./segmentation.py -a 1m -p 32m
```

如果你想设置得更具体，你可以用 --b0、--l0、--b1 和 --l1 寄存器自己设置基址寄存器和界限寄存器的值。试试看。

最后，你可以随时运行：

```
提示符> ./segmentation.py -h
```

以获得完整的标志和选项列表。
