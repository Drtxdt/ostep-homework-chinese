# 概述

这个有趣的小作业测试你是否理解多级页表的工作原理。是的，对于“有趣”这个词在前一句中的使用，可能存在一些争议。这个程序叫做：`paging-multilevel-translate.py`

一些基本假设：

- 页大小是一个不切实际的小值：32 字节
- 相关进程（假设只有一个）的虚拟地址空间是 1024 页，即 32 KB
- 物理内存由 128 页组成

因此，一个虚拟地址需要 15 位（5 位用于偏移量，10 位用于 VPN）。

一个物理地址需要 12 位（5 位偏移量，7 位用于 PFN）。

系统采用多级页表。因此，虚拟地址的高 5 位用于索引页目录；页目录项 (PDE) 如果有效，则指向页表的一个页。每个页表页包含 32 个页表项 (PTE)。每个 PTE 如果有效，则保存所讨论虚拟页的所需转换（物理帧号，或 PFN）。

因此，PTE 的格式如下：

```
有效位 | PFN6 ... PFN0
```

因此是 8 位或 1 字节。

PDE 的格式基本相同：

```
有效位 | PT6 ... PT0
```

首先，你获得两条信息。

首先，给你页目录基址寄存器 (PDBR) 的值，它告诉你页目录位于哪个页上。

其次，给你每个内存页的完整转储。一个页的转储如下所示：

```
页 0: 08 00 01 15 11 1d 1d 1c 01 17 15 14 16 1b 13 0b ...
    页 1: 19 05 1e 13 02 16 1e 0c 15 09 06 16 00 19 10 03 ...
    页 2: 1d 07 11 1b 12 05 07 1e 09 1a 18 17 16 18 1a 01 ...
    ...
```

这显示了在第 0、1、2 页等页上找到的 32 个字节。第 0 页的第一个字节（第 0 字节）的值为 0x08，第二个是 0x00，第三个是 0x01，依此类推。

然后，给你一个要转换的虚拟地址列表。

使用 PDBR 找到这个虚拟页相关的页表项。然后查看它是否有效。如果有效，使用转换来形成最终的物理地址。使用这个地址，你可以找到内存引用所查找的**值**。

当然，虚拟地址可能无效，从而产生故障。

一些有用的选项：

```
-s 种子, --seed=种子       随机种子
  -n 数量, --addresses=数量   要生成的虚拟地址数量
  -c, --solve                为我计算答案
```

像往常一样，更改种子以获得不同的问题。

更改生成的虚拟地址数量，以便为给定的内存转储进行更多的转换。

使用 -c（或 --solve）来显示解决方案。
