# 概述

在本作业中，你将使用一个简单的程序，名为 paging-linear-translate.py，来检验你是否理解使用线性页表时简单的虚拟地址到物理地址转换如何工作。要运行该程序，请记住要么只输入程序名称 (./paging-linear-translate.py)，或者可能这样输入 (python paging-linear-translate.py)。当你使用 -h（帮助）标志运行它时，你会看到：

```
提示符> ./paging-linear-translate.py -h
用法: paging-linear-translate.py [选项]

选项:
  -h, --help              显示此帮助信息并退出
  -s SEED, --seed=SEED    随机种子
  -a ASIZE, --asize=ASIZE
                          地址空间大小 (例如: 16, 64k, ...)
  -p PSIZE, --physmem=PSIZE
                          物理内存大小 (例如: 16, 64k, ...)
  -P PAGESIZE, --pagesize=PAGESIZE
                          页大小 (例如: 4k, 8k, ...)
  -n NUM, --addresses=NUM
                          要生成的虚拟地址数量
  -u USED, --used=USED    地址空间中已使用的百分比
  -v                      详细模式
  -c                      为我计算答案
```

首先，不带任何参数运行程序：

```
提示符> ./paging-linear-translate.py
ARG 种子 0
ARG 地址空间大小 16k
ARG 物理内存大小 64k
ARG 页大小 4k
ARG 详细模式 False

页表的格式很简单：
最高位（最左位）是有效位。
  如果该位是 1，条目的其余部分是物理帧号 (PFN)。
  如果该位是 0，则该页无效。
如果你想在每个页表条目旁边打印 VPN 编号，请使用详细模式 (-v)。

页表 (从条目 0 向下到最大大小)
   0x8000000c
   0x00000000
   0x00000000
   0x80000006

虚拟地址跟踪
  VA  0: 0x00003229 (十进制:    12841) --> 物理地址 或 无效？
  VA  1: 0x00001369 (十进制:     4969) --> 物理地址 或 无效？
  VA  2: 0x00001e80 (十进制:     7808) --> 物理地址 或 无效？
  VA  3: 0x00002556 (十进制:     9558) --> 物理地址 或 无效？
  VA  4: 0x00003a1e (十进制:    14878) --> 物理地址 或 无效？
```

对于每个虚拟地址，写下它转换到的物理地址，或者写下它是一个越界地址（例如，段错误）。

如你所见，程序为你提供的是特定进程的页表（请记住，在具有线性页表的真实系统中，每个进程有一个页表；这里我们只关注一个进程、它的地址空间，因此是一个页表）。页表告诉你，对于地址空间的每个虚拟页号 (VPN)，该虚拟页映射到特定的物理帧号 (PFN) 从而有效，或者无效。

页表条目的格式很简单：最左边（最高位）是有效位；如果有效位是 1，则其余位是 PFN。

在上面的例子中，页表将 VPN 0 映射到 PFN 0xc（十进制 12），VPN 3 映射到 PFN 0x6（十进制 6），并将另外两个虚拟页 1 和 2 标记为无效。

因为页表是一个线性数组，上面打印的内容是如果你自己查看内存位会看到的内容的副本。然而，有时使用详细标志 (-v) 运行此模拟器会更方便；此标志还会打印出页表中的 VPN（索引）。以上面的例子为例，使用 -v 标志运行：

```
页表 (从条目 0 向下到最大大小)
  [       0]   0x8000000c
  [       1]   0x00000000
  [       2]   0x00000000
  [       3]   0x80000006
```

那么，你的任务就是使用这个页表将跟踪中给定的虚拟地址转换为物理地址。让我们看第一个：VA 0x3229。要将此虚拟地址转换为物理地址，我们首先必须将其分解为其组成部分：虚拟页号 (VPN) 和偏移量 (offset)。我们通过记录地址空间大小和页大小来做到这一点。在这个例子中，地址空间设置为 16KB（一个非常小的地址空间），页大小为 4KB。因此，我们知道虚拟地址中有 14 位，偏移量是 12 位，剩下 2 位用于 VPN。因此，对于我们的地址 0x3229，它是二进制 11 0010 0010 1001，我们知道最高两位指定 VPN。因此，0x3229 位于虚拟页 3 上，偏移量为 0x229。

接下来，我们查看页表，看 VPN 3 是有效的并映射到某个物理帧，还是无效的，我们确实看到它是有效的（最高位是 1）并映射到物理页 6。因此，我们可以通过取物理页 6 并与偏移量组合来形成最终的物理地址，如下所示：0x6000（物理页，移位到适当位置）与 0x0229（偏移量）进行 OR 运算，得到最终的物理地址：0x6229。因此，我们可以看到在这个例子中，虚拟地址 0x3229 转换为物理地址 0x6229。

要查看其余解决方案（在你自己计算之后！），只需使用 -c 标志运行（一如既往）：

```
...
VA  0: 00003229 (十进制: 12841) --> 00006229 (25129) [VPN 3]
VA  1: 00001369 (十进制:  4969) --> 无效 (VPN 1 无效)
VA  2: 00001e80 (十进制:  7808) --> 无效 (VPN 1 无效)
VA  3: 00002556 (十进制:  9558) --> 无效 (VPN 2 无效)
VA  4: 00003a1e (十进制: 14878) --> 00006a1e (27166) [VPN 3]
```

当然，你可以更改这些参数中的许多参数来创建更有趣的问题。运行程序时使用 -h 标志以查看有哪些选项：

- -s 标志更改随机种子，从而生成不同的页表值以及要转换的不同虚拟地址。
- -a 标志更改地址空间的大小。
- -p 标志更改物理内存的大小。
- -P 标志更改页的大小。
- -n 标志可用于生成更多要转换的地址（而不是默认的 5 个）。
- -u 标志更改有效映射的比例，从 0% (-u 0) 到 100% (-u 100)。默认是 50，这意味着虚拟地址空间中大约 1/2 的页将是有效的。
- -v 标志打印出 VPN 编号以使你的工作更轻松。
