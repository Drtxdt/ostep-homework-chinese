# 概述

欢迎使用 `ssd.py`，这是 OSTEP 作者免费提供给你的又一个精彩模拟器，OTEP 本身也是免费的。很快，你就会认为生活中所有重要的东西都是免费的！而且，事实证明，某种程度上确实如此：你呼吸的空气，你给予和接受的爱，以及一本关于操作系统的书。你还需要什么呢？

要运行模拟器，你只需像往常一样：

```
prompt> ./ssd.py
```

这个模拟器模拟了几种不同类型的 SSD。第一种是我们称之为"理想"的 SSD，实际上它根本不太像 SSD；它更像是一个完美的内存。要模拟这个 SSD，输入：

```
prompt> ./ssd.py -T ideal
```

为了了解这个如何工作，让我们创建一个简单的工作负载。对于 SSD 来说，工作负载只是一系列发送到设备的低级 I/O 操作。ssd.py 支持三种操作：读取（接收一个地址来读取，并返回数据）、写入（接收一个地址和一段要写入的数据，这里是一个字母）和修剪（接收一个地址）。修剪操作用于指示先前写入的块不再存活（例如，它所在的文件已被删除）；这对于基于日志的 SSD 特别有用，它可以在垃圾回收期间回收该块的空间，并在 FTL 中释放空间。让我们运行一个只包含一次写入的简单工作负载：

```
prompt> ./ssd.py -T ideal -L w10:a -l 30 -B 3 -p 10
```

`-L`标志允许我们指定一个以逗号分隔的命令列表。这里，为了写入逻辑页 10，我们包含命令 "w10:a"，意思是"写入"到逻辑页 "10" 数据 "a"。我们还用标志 `-l 30 -B 3 -p 10`包含了关于 SSD 大小的一些其他细节，但让我们暂时忽略这些。

运行上述命令后，你应该在屏幕上看到：

```
FTL   (空)
块 0          1          2
页  0000000000 1111111111 2222222222
      0123456789 0123456789 0123456789
状态 iiiiiiiiii iiiiiiiiii iiiiiiiiii
数据
存活


FTL    10: 10
块 0          1          2
页  0000000000 1111111111 2222222222
      0123456789 0123456789 0123456789
状态 iiiiiiiiii viiiiiiiii iiiiiiiiii
数据             a
存活             +
```

第一块信息显示了 SSD 的初始状态，第二块显示了其最终状态。让我们逐一讲解每个部分，以确保你理解它们的含义。

每块输出的第一行显示 FTL 的内容。这个模拟器只模拟一个简单的页映射 FTL；因此，其中的每个条目显示了任何存活数据项的逻辑到物理页映射。

在初始状态中，FTL 是空的：

```
FTL   (空)
```

然而，在最终状态中，你可以看到 FTL 将逻辑页 10 映射到物理页 10：

```
FTL    10: 10
```

这种简单映射的原因是我们正在运行"理想" SSD，它实际上就像一个内存；如果你写入逻辑页 X，这个 SSD 会（神奇地）将数据写入物理页 X（实际上，你甚至根本不需要 FTL；我们只使用理想的 SSD 来显示真正的 SSD 在擦除和数据复制方面做了多少额外工作，与之相比理想的内存）。

输出的下一行只是标记底层 Flash 的块和物理页，模拟器正在建模这些 Flash：

```
块 0          1          2
页  0000000000 1111111111 2222222222
      0123456789 0123456789 0123456789
```

在这个模拟中，你可以看到 SSD 有 3 个物理 Flash 块，并且每个块有 10 个物理页。每个块都有编号 (0, 1, 和 2)，每个页也有编号 (从 00 到 29)；为了使显示紧凑（宽度方向），页编号显示在两行上。因此，物理页 "10" 在第一行标记为 "1"，在第二行标记为 "0"。

下一行显示每个页的状态，即根据章节，它是无效 (i)、已擦除 (E) 还是有效 (v)：

```
状态 iiiiiiiiii viiiiiiiii iiiiiiiiii
```

"理想" SSD 的状态有点奇怪，因为在一个块中可以混合 "v" 和 "i"，并且该块永远不会是 "E"（已擦除）。下面，使用更现实的"直接"和"日志" SSD 时，你也会看到 "E"。

最后两行显示任何被写入页的"内容"（在"数据"行）以及该数据当前是否存活（即在 FTL 中引用）在"存活"行：

```
块 0          1          2
页  0000000000 1111111111 2222222222
      0123456789 0123456789 0123456789
数据             a
存活             +
```

这里，我们可以看到在块 2（即页 10）上，有数据 "a"，并且它确实是存活的（用 "+" 符号表示）。

在讨论更现实的 SSD 类型之前，让我们稍微扩展一下工作负载。在写入数据之后，让我们读取它，然后使用修剪来删除它：

```
prompt> ./ssd.py -T ideal -L w10:a,r10,t10 -l 30 -B 3 -p 10
```

如果你运行这个，你会看到两个相同的状态：初始（空）状态和最终（也是空的！）状态。不太令人兴奋！要了解更多情况，你必须使用更多的标志。是的，这个 SSD 模拟器使用了很多标志；抱歉，所有喜欢简洁的人！但可惜，这里有一些复杂性我们必须探索。

一个有用的标志是 `-C`，它只显示发出的每个命令，以及是否成功。

```
prompt> ./ssd.py -T ideal -L w10:a,r10,t10 -l 30 -B 3 -p 10 -C

数据             a
存活

prompt>
```

在这里，你可以看到写入、读取和修剪，并且你还可以看到每个命令返回了什么：分别是成功、读取的数据和成功。这在之后模拟器随机生成操作时会更有趣。

当然，你可以同时使用 `-C`和 `-F`来显示所有内容（这个练习留给读者）。

模拟器还允许你生成随机工作负载，而不是自己指定操作。使用 "-n" 标志，并附带一个数字（我们还用 "-s" 指定一个随机种子以获得特定的工作负载）：

```
prompt> ./ssd.py -T ideal -l 30 -B 3 -p 10 -n 5 -s 10
```

如果你用 `-C`、`-F`或两者一起运行，你会看到确切的命令、Flash 的中间状态，或两者。但是，你也可以使用 "-q" 标志来测验你认为命令是什么。因此，运行以下命令：

```
prompt> ./ssd.py -T ideal -l 30 -B 3 -p 10 -n 5 -s 10 -q
(为简洁起见，输出省略)
```

现在，通过检查中间状态，看看你是否能辨别出命令必须是什么（写入和修剪被完全未指定，而读取只是要求你找出返回了哪些数据）。

然后，你可以手动使用 `-C -F`来显示所有内容，或者只需添加 `-c`标志来"解决"问题，以检查你的答案。

现在让我们做同样的事情（随机生成五个操作的工作负载），但使用不同的更现实的 SSD。第一个是章节中提到的"直接" SSD。这也不是特别现实，但至少使用了擦除和编程来更新 Flash。具体来说，当一个逻辑页被写入时，它被直接映射到相同编号的物理页。这种映射首先需要读取该块中的所有存活数据，然后擦除该块，然后进行一系列编程以恢复所有先前存活的数据，并将新数据写入 Flash。让我们运行它，显示命令 (-C) 但不显示中间状态 (不使用 -F)：

```
prompt> ./ssd.py -T direct -l 30 -B 3 -p 10 -n 5 -s 10 -C

FTL   (空)
块 0          1          2
页  0000000000 1111111111 2222222222
      0123456789 0123456789 0123456789
状态 iiiiiiiiii iiiiiiiiii iiiiiiiiii
数据
存活

命令 0:: write(12, z) -> 成功
命令 1:: write(19, 9) -> 成功
命令 2:: write(9, f) -> 成功
命令 3:: trim(9) -> 成功
命令 4:: read(19) -> 9

FTL    12: 12  19: 19
块 0          1          2
页  0000000000 1111111111 2222222222
      0123456789 0123456789 0123456789
状态 EEEEEEEEEv EEvEEEEEEv iiiiiiiiii
数据           f   z      9
存活               +      +

prompt>
```

正如你从最终状态看到的，FTL 包含两个存活映射：逻辑页 12 指向物理页 12，逻辑页 19 指向物理页 19（记住，这是直接映射）。你还可以看到三个包含信息的数据页：物理页 9 包含 "f"，12 包含 "z"，19 包含 "9"（数据可以是字母、数字或任何单个字符）。但是，也要注意 "9" 已被修剪；这从 FTL 中删除了它的条目，但数据仍然在那里休眠（目前）。如果你随后尝试读取逻辑页 9，它将不再成功：

```
prompt> ./ssd.py -T direct -l 30 -B 3 -p 10 -C -L w9:f,t9,r9

      0123456789 0123456789 0123456789
```

我们应该关注的最后一个 SSD 是实际上最现实的一个，它使用日志结构化（就像大多数真正的 SSD 一样）。要使用它，只需将 SSD 类型更改为 "log"（我们再次打开 -C，这样我们就可以知道哪些操作发生了，而不是测验自己）：

```
prompt> ./ssd.py -T log -l 30 -B 3 -p 10 -s 10 -n 5 -C

FTL   (空)
块 0          1          2
页  0000000000 1111111111 2222222222
      0123456789 0123456789 0123456789
状态 iiiiiiiiii iiiiiiiiii iiiiiiiiii
数据
存活

命令 0:: write(12, z) -> 成功
命令 1:: write(19, 9) -> 成功
命令 2:: write(9, f) -> 成功
命令 3:: trim(9) -> 成功
命令 4:: read(19) -> 9

FTL    12:  0  19:  1
块 0          1          2
页  0000000000 1111111111 2222222222
      0123456789 0123456789 0123456789
状态 vvvEEEEEEE iiiiiiiiii iiiiiiiiii
数据  z9f
存活  ++

prompt>
```

注意日志结构化 SSD 如何将数据写入 Flash。首先，当前日志块（在这种情况下是块 0）被擦除。然后，页按顺序编程。使用 -F 标志查看每个步骤的更多细节。

模拟器还可以显示更多统计数据，包括操作计数和所建模 SSD 完成给定工作负载的估计时间。要查看这些，使用 -S 标志：

```
prompt> ./ssd.py -T log -l 30 -B 3 -p 10 -s 10 -n 5 -S

(省略部分内容)

每个块的物理操作
擦除   1          0          0          总计: 1
写入   3          0          0          总计: 3
读取   1          0          0          总计: 1

逻辑操作总计
  写入计数 3 (0 失败)
  读取计数 1 (0 失败)
  修剪计数 1 (0 失败)

时间
  擦除时间 1000.00
  写入时间 120.00
  读取时间 10.00
  总时间 1130.00
```

在这里，你可以看到每个块的物理擦除、写入和读取以及每个的总和，然后是发送到设备的逻辑写入、读取和修剪的数量，最后是估计的时间。你可以分别使用 -R、-W 和 -E 标志更改低级操作（如读取、编程和擦除）的成本。

最后，在 SSD 处于日志结构化模式时，可以配置一个垃圾回收器 (GC) 定期运行。此行为由 -G 和 -g 标志控制，它们设置确定垃圾回收器是否应运行的高水位线和低水位线。将高水位线设置为值 N（即 -G N）意味着当 GC 注意到有 N 个块正在使用时，它应该运行。将低水位线设置为 M（即 -g M）意味着 GC 应该运行直到只有 M 个块在使用。

这里的 -J 标志也很有用：它显示 GC 发出的低级命令（存活数据的读取和写入，然后是回收块的擦除）。下面发出 60 个操作，并将高水位线和低水位线分别设置为 3 和 2。

```
prompt> ./ssd.py -T log -l 30 -B 3 -p 10 -s 10 -n 60 -G 3 -g 2 -C -F -J
```

使用 `-C`、`-F`和 `-J`可以让你真正看到日志结构化模拟内部发生的每一步。

还有其他几个值得了解的标志。一直以来，我们一直在使用以下三个标志来控制模拟 SSD 的大小：

```
-l NUM_LOGICAL_PAGES, --logical_pages=NUM_LOGICAL_PAGES  接口中的逻辑页数量
  -B NUM_BLOCKS, --num_blocks=NUM_BLOCKS                   SSD 中的物理块数量
  -p PAGES_PER_BLOCK, --pages_per_block=PAGES_PER_BLOCK    每个物理块的页数
```

你可以更改这些值来模拟比我们目前模拟的简单 SSD 更大或更小的 SSD。

另一组控件让你可以更精确地控制随机生成的工作负载。`-P`标志让你控制读取/写入/修剪出现的概率。例如，使用 `-P 30/35/35`意味着大约 30% 的操作是读取，35% 是写入，35% 是修剪。

`-r`标志允许读取被发送到非存活地址（默认情况下只对存活数据发出读取）。因此，`-r 10`意味着大约 10% 的读取会失败。

最后，`-K`和 `-k`标志让你为工作负载添加一些"偏斜"。偏斜首先由 `-K`指定，例如，`-K 80/20`使 80% 的写入针对 20% 的逻辑空间（一种热/冷类型的工作负载）。偏斜在真实工作负载中很常见，并且对垃圾回收等有不同的影响，因此能够对其进行建模是很好的。相关的 `-k`标志让你指定偏斜开始的时间；具体来说，`-k 50`意味着在 50 次写入后，开始进行偏斜（在此之前，写入将从逻辑空间的所有可能页中随机选择）。
